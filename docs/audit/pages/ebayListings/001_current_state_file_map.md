**Date:** 2026-05-12  
**Scope:** Admin eBay Listings page discovery/audit only. No code changes.  
**Page:** `pages/admin/ebay-listings.html`

---

## 1. Executive map

The current eBay Listings page is a vanilla HTML + modular JS admin page. It is not a reporting workspace yet. It is primarily a listing operations console powered by:

- `products` as the local source of truth for shop/catalog product identity and eBay linkage fields.
- Direct Supabase client reads from `products`, `product_gallery_images`, and `product_variants`.
- Edge functions for live eBay Inventory/Account/Marketing/Taxonomy/OpenAI calls.
- Local product-row fields as the only persistent listing state visible on the page.

The current architecture is good for create/edit/publish/end workflows, but weak for sales/performance/fee/profit decisions because those datasets exist elsewhere and are not joined into this page.

## 4. Frontend data flow

### Initial load

1. `init()` runs.
2. `initAdminNav("eBay Listings")` mounts admin nav.
3. `initFooter()` mounts footer.
4. `requireAdmin()` gates access.
5. `setView(currentView)` picks cards on mobile / table on desktop.
6. `loadProducts()` queries Supabase.
7. `loadPoliciesCache()` calls `ebay-manage-listing` with `action: "get_policies"`.

### Main Supabase query

`loadProducts()` reads from `products`:

```text
id, code, name, slug, price, weight_g,
catalog_image_url, catalog_hover_url, primary_image_url, is_active,
ebay_sku, ebay_offer_id, ebay_listing_id, ebay_status,
ebay_category_id, ebay_price_cents, ebay_item_group_key,
ebay_volume_promo_id, ebay_store_category,
product_gallery_images(url, position, is_active),
product_variants(id, option_name, option_value, stock, preview_image_url, sort_order, is_active)
```

No sales, order, fee, profit, traffic, watcher, impression, click, or conversion data is loaded by this page today.

## 6. `ebay-manage-listing` actions relevant to the page

The main page calls the following `ebay-manage-listing` actions:

| Action | eBay API family | Page workflow |
|---|---|---|
| `get_policies` | Account API | Setup panel and policy dropdown population |
| `setup_location` | Inventory API | Setup panel location creation/verification |
| `get_item` | Inventory API | Resume draft, edit single listing, edit variant items |
| `get_item_group` | Inventory API | Edit group/variant listings |
| `get_offers` | Inventory API | Edit price/policy info; variant offer discovery |
| `create_item` | Inventory API | Push Step 1 single or variant SKU inventory items |
| `update_item` | Inventory API | Edit listing inventory item data |
| `create_item_group` | Inventory API | Push Step 2 variant item group |
| `update_item_group` | Inventory API | Edit variant group metadata |
| `create_offer` | Inventory API | Push Step 2 single item offer |
| `create_group_offer` | Inventory API | Push Step 2 group offers for variants |
| `update_offer` | Inventory API | Edit price/quantity/policies/store category |
| `publish` | Inventory API | Publish single offer |
| `publish_group` | Inventory API | Publish variant group |
| `withdraw` | Inventory API | End single listing |
| `withdraw_group` | Inventory API | End variant/group listing |
| `bulk_update` | Inventory API | Bulk price/quantity update |
| `create_volume_discount` | Marketing API | Create volume pricing promotion |
| `get_volume_discount` | Marketing API | Load existing volume promotion into edit modal |
| `update_volume_discount` | Marketing API | Update volume pricing tiers |
| `delete_volume_discount` | Marketing API | Remove volume pricing promotion |

`ebay-manage-listing` also has webhook setup actions not currently exposed by the visible HTML except through possible manual/API usage.

## 11. Unresolved unknowns

| Unknown | Why unknown |
|---|---|
| Whether eBay traffic/performance APIs are deployed anywhere | Search found docs but no active edge function/page wiring for impressions/clicks/watchers/conversion. |
| Whether `ebay_category_cache.category_name` is meaningful | `ebay-taxonomy` stores `Category ${categoryId}` rather than true category names for aspect cache rows. |
| Whether all eBay OAuth scopes needed for Marketing promoted-listing campaigns are currently requested | `_shared/ebayUtils.ts` refresh scope list includes inventory/fulfillment/finances/account, but the current listed scope set does not visibly include `sell.marketing`; existing volume discount calls rely on Marketing API and may depend on token scope history/config. |
| Whether active listings can be reliably joined to sold order history by SKU/listing ID | `ebay-sync-orders` writes orders/line items, but this audit did not find a page-level mapping view from `products.code` / `ebay_sku` / variant SKU to eBay order line history. |
| Whether line item rows store eBay listing ID | Search indicates order sync is product/title/order based; no direct listing-ID field surfaced in the page query. A join view may be needed. |
